Responsive Redesign

Original Pancake House Website

For this project, I chose the redesign the website of one of my favorite restaurants in my hometown.

The Original Pancake House is a well known national franchise with over 100 locations nationwide, so I expected their website to be very fleshed out.

I was shocked to find that it was quite difficult to use and interpret. I chose to redesign this website and began by identifying usability and accessibility problems.

Original Website

Usability Problems

I began by idenfying design problems in four main areas; usability, memorability, learnability, and efficiency. My findings are summarized here:

Usability

    High contrast colors (all bright primary colors) and strange grouping can overwhelm the user when they first look at it
    No adaptabilities for different sizes of screens and does not resize well
    Too many different typefaces and layout styles confuse the user

Memorability

    Hard to interpret information because it is not grouped or clearly sectioned
    The cluttered top contrasted with the excess white space towards the end of the webpage overwhelms and confuses the user
    There isnâ€™t any one element (name, logo, slogan) that clearly stands out to the user when they first look at the page

Learnability

    Inconsistent use of typefaces makes it difficult to comprehend the content and see the hierarchy
    Lots of long descriptions with only two (bad quality) images makes it hard to remember and interpret information
    The lack of a clear hierarchy makes it difficult to to know which information is being presented on the page and where to look for it

Efficiency

    Lots of repetition in content across the different pages of the website which could confuse the user on where to look
    Confusing to nagivate the webpage because there is a top menu and a left menu
    Inconsistent and uneven spacing between elements and too much white space makes it hard to look for information

Accessibility Issues

By using WebAIM WAVE to identify accessibility concerns, I found contrast errors found on the homepage where there is very little contrast between some of the text and the background. The page was also missing many heading levels and overall structural components. I also used a screenreader to examine more accesibility concerns and found that much of the homepage's main content was not recognized by the screenreader. This was probably due to the fact that many heading levels were missing on this webpage (as detected by WebAIM WAVE) and that the page was lacking an overall structure that made it easier to read and detect. Ultimately, these finding were very concerning because it demonstrates that the original website has nearly no adaptability and is very inaccessible.

Prototyping

Next I created simple wireframe prototypes of a potential new interface for the homepage of this website. I tried to remedy many of the design issues I saw above in these prototypes. For example, I established clear sections and grouped information into these sections so that the webpage would be easier to read and parse and so that it was more visually appealing. More information on how I chose to fix the issues that were identified can be found directly on the prototypes themselves. The lofi prototypes are a rough first look of the structure of the webpage with placeholders for many of the elements. I created these prototypes in three different screen sizes (website, tablet, and mobile) to better demonstrate how the page will adapt as the screen size shrinks and grows.

Once I finished these lofi prototypes, I moved onto creating more high fidelity prototypes that would illustrate how these designs could be turned into an actual webpage. These were based off of the structure outlined in the lofi prototypes, but I filled in many of the missing elements and added important stylistic choicse. In these prototypes, I annotated significant design choices and their implementation so that they could be easily translated into a real webpage by any designer. Similar to the lofi prototypes, I created three screen size mockups.

Style Guide

Next, I created a style guide to reflect the design choices and interactive states in my design. This design guide serves as a useful tool for both myself and other potential designs. It ensures that I remain consistent in my implementation and designs and that there is a unified and coherent experience in the webpage, but it also is useful because it would allow any other designer to "pick up where I left off" and redesign other pages of the website using a similar style to the redesigned homepage.

Finally I created the new homepage!

Ultimately, I found this project to be really fun and rewarding! It was great to examine an existing website and to be able to critique it and learn about which formal principles of design it violated. It was also really great to think about how I could improve this website on my own and how I organize the information that it presented.
